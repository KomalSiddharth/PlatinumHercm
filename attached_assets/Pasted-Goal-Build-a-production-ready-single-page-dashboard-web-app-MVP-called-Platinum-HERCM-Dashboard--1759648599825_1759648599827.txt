Platinum Streak detection:

Criteria come from Excel file; parse and evaluate. Default criteria if Excel missing:

4 consecutive weeks where:

For each week: hercm filled (non-empty) AND checklist_completed==true

Ritual completion rate ≥ 80% across the 4 weeks

Implementation:

Cron verifies each user weekly: maintain streak_count or compute sliding window.

When user achieves 4-week streak, create achievements row, award platinum_points (e.g., +500), issue badge and notify user (in-app toast and email).

API endpoints (must implement exactly)

POST /api/auth/signup → {name,email,password}

POST /api/auth/login → returns JWT

GET /api/hercm/week?week_start=YYYY-MM-DD → returns user hercm entry

POST /api/hercm → create/update current week entry (server validates +1 rule)

POST /api/checklist/:entryId/complete → body: { item_index } → server records checklist_progress with server timestamp

GET /api/rituals POST /api/rituals PATCH /api/rituals/:id POST /api/rituals/:id/complete → returns ledger update

GET /api/leaderboard?period=week|month → returns leaderboard

GET /api/courses POST /api/courses PATCH /api/user_courses/:id

GET /api/recommendations → based on recent hercm entries

GET /api/admin/users/:id/activity → admin-only detailed audit

POST /api/admin/award → admin awards points/badge manually

POST /api/cron/rollover → manual trigger for weekly rollover (protected by env key)
